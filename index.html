Features

Everything you need, nothing you don't.

🌐

Multi-language parsing

tree-sitter extracts symbols and docstrings from Python, JS/TS, Go, Rust, Java, C/C++, and more.

🧠

Intent classification

Weighted scoring detects whether you want to implement, debug, refactor, test, explain, review, or search — and picks the right retrieval strategy.

🔗

MCP native

One command writes .claude/mcp.json, .cursor/mcp.json, and .vscode/settings.json automatically.

📊

Token budget

tiktoken-accurate counting keeps context within your budget. Detail level 0–3 trims output gracefully.

📄

Git temporal context

Per-symbol git author and date so the AI assistant knows which code is recent and who owns it.

🔒

Concurrent-safe indexing

Lock file prevents concurrent writes. 2-second debounce stops redundant re-indexing on save hooks.

🏠

Local-first

SQLite index lives in .ctx/. No cloud calls, no embedding API keys required.

📋

Project memory

Store rules, references, and notes that get injected into context automatically on every query.

Architecture

Simple pipeline from CLI to AI assistant.

CLI (semtree)
     |
     v
Indexer (coordinator.py)
  walk → SHA-1 hash → tree-sitter parse → extract symbols → git blame
     |
     v
SQLite (.ctx/index.db)
  files | symbols (FTS5) | memory
     |
     v
Retrieval (retrieval/)
  intent classifier → search.py → policy.py
     |
     v
Context Builder (context/builder.py)
  budget.py + levels.py → Markdown output
     |
     v
MCP Server (mcp.py)
  index_project | get_context | search_symbols
     |
     v
AI Assistant (Claude Code / Cursor / Copilot / Codex)

CLI commands

Everything you can do from the terminal.

semtree indexIndex the project (incremental by default)
semtree index --forceRe-index all files, ignoring cache
semtree context "QUERY"Build context for a task, print to stdout
semtree context "QUERY" -b 4000Limit context to 4,000 tokens
semtree context "QUERY" -l 0Override detail level (0=minimal, 3=full)
semtree search "QUERY"Search symbols by name or keyword
semtree search "QUERY" -k classFilter by kind: function|class|method|const|type|var
semtree search "QUERY" --jsonOutput results as JSON
semtree statusShow index stats (files, symbols, last updated)
semtree memory add rule KEY VALStore a project rule in the index
semtree memory listList all memory entries
semtree setup --target allConfigure all AI assistants
semtree setup --dry-runPreview setup changes without writing
semtree config --initWrite default config to .ctx/semtree.json
